# TravelPoints v1.1 Update Plan

**Status:** Planning
**Date:** 2026-04-06
**Authors:** Charlie (product), Lewis (tech)

---

## Overview

Two-stage update to broaden the region/province system and add a global "visited regions" map mode.

- **Stage 1:** Expand province coverage to 30+ more countries (area-based Tier 2 promotion), make cities explicitly checkable everywhere, and show cities on province maps.
- **Stage 2:** Add a "visited regions" map mode to the global world map.

---

## Stage 1A — Tier 2 Expansion (Area-Based)

### What changes

Countries at global area rank 11–40 that are currently Tier 3 get formally reclassified as **Tier 2** in `points.js`. They gain:
- Province/region maps on their country detail page
- Province-based explorer points (same formula as existing Tier 2)
- City overlay on their province map (see Stage 1C)

Countries already in Tier 1 or Tier 2 within those ranks are unaffected.

## Stage 1B — City Checking for Tier 3

### What changes

The current Tier 3 city exploration is invisible: the backend silently picks the top 15 cities by population and calculates an exploration ratio. Users never see or interact with the city list for scoring purposes.

**New behaviour:**
- Each T3 country gets a defined set of cities in the seed (see city count rules below)
- Cities are shown as checkboxes on the country detail page
- Checking a city earns explorer points proportional to that city's share of the total city pool's population
- The existing invisible top-15 calculation is replaced by: `(visited_city_pop / total_seeded_city_pop) × explorer_ceiling`

### City count rules

| Country group | Cities |
|--------------|--------|
| Microstates (VA, MC, SM, LI, AD, SG) | 0 |
| Very small nations (area < ~10,000 km² or pop < 500k) | 2 |
| Standard T3 nations | 3 |
| Top 60 countries by area (T3 only; T1/T2 unchanged) | 6 |

The top 60 by area threshold falls at roughly **476,000 km²** (between Cameroon and Thailand). Applying this to T3 only, the countries that get 6 cities are: CA, AU, AR, KZ, DZ, SA, SD, LY, MN, PE, TD, NE, AO, ML, BO, MR, VE, NA, MZ, CL, ZM, AF, SS (South Sudan), SO (Somalia), CF (Central African Republic), UA (Ukraine), MG (Madagascar), BW (Botswana), TM (Turkmenistan), CM (Cameroon).

Note: once a country is promoted to T2 (Stage 1A), its city count follows the T2 rules (existing counts, unchanged), not this rule.

### Data work required

Most T3 countries currently have no cities in the DB. We need to seed cities for all ~165 T3 countries (after Stage 1A promotions, ~148 remaining). This is significant data entry but formulaic — top cities by population are well-documented.

## Stage 1C — City Checking for Tier 2

### What changes

Tier 2 countries currently show province maps but have no city interaction. Cities appear in the DB for some T2 countries but are never surfaced to the user.

**New behaviour:**
- Cities for T2 countries are shown as checkable items on the country detail page
- Visiting a city contributes to explorer points **on top of** the province-based score
- City contribution uses a **0.4 multiplier** relative to the province formula

### Scoring formula for T2 cities

The 0.4 multiplier means:
```
city_explorer_points = (visited_city_pop / total_seeded_city_pop) × explorer_ceiling × 0.4
```

Cities contribute up to 40% of the explorer ceiling on top of province exploration. Total explorer points for T2 = province points + city points (capped at explorer ceiling).

**Open question:** should the combined province + city points be capped at the explorer ceiling, or can cities push you slightly above it? Recommend capping at ceiling for consistency.

### City counts for T2

T1/T2 city counts stay as currently defined — we are **not** changing these. Countries already in the DB with city data keep their current counts.

## Stage 1D — Cities Visible on Province Maps

### What changes

Currently the ProvinceMap component shows only province outlines. Cities exist only as a scrollable list below the map.

**New behaviour:**
- City dots are overlaid on the ProvinceMap SVG
- Hovering a province region highlights which cities fall within it
- Clicking a province region opens an inline panel showing its cities as checkboxes (alongside the existing province toggle)
- Checking a city from this panel is equivalent to checking it in the list

### Technical approach

Cities need lat/lng coordinates added to the DB (currently cities have no coordinates — only name and population). We need to:
1. Add `lat` and `lng` columns to the `cities` table (new migration)
2. Add coordinates to city seed data
3. In the ProvinceMap component, render city dots using the country's d3-geo projection
4. On province hover, do a point-in-polygon check against the province GeoJSON feature to show its cities

**Scope for v1.1:** US and China explicitly called out as priority (Beijing/Shanghai should appear on the China province map; NYC/LA/Chicago etc. on the US province map).

## Stage 1E — Quick-Add from World Map

### What changes

Currently clicking a country on the world map navigates to the full country detail page (`/countries/{code}`). If you just want to log a visit, you have to load the whole page.

**New behaviour:**
- Clicking a country on the world map opens a lightweight popover/drawer
- Shows: country name, flag, base points, "Mark as visited" / "Remove visit" button
- "View full details →" link still takes you to the detail page

### Technical approach

- The Map.jsx click handler currently calls `navigate(`/countries/${code}`)` — change to open a modal/popover instead
- Fetch `/api/countries/{code}?home_country={home}` for the basic score data
- POST/DELETE to `/api/users/{id}/countries` to add/remove the visit
- Keep the navigation link for users who want provinces/cities

## Stage 2 — Visited Regions Map Mode

### What changes

A new mode toggle on the global world map (alongside the existing "My Map" / "Explore" modes) called **"Regions"**.

In this mode:
- For countries with province data (all current T1/T2, plus new T2 from Stage 1A): the map renders province-level boundaries instead of country outlines, coloured by visited status
- For countries without province data (T3): the country renders as a solid block, coloured if visited (same as "My Map" mode)
- City markers (dots) are optionally overlaid for countries where the user has visited cities

### Technical approach

The current world map uses `react-simple-maps` with a single country-level TopoJSON (`world-atlas` 110m). The regions mode needs a different strategy:

**Option A — Per-country GeoJSON overlay:**
When a T1/T2 country is visited, fetch its `/geo/{code}.json` and render its provinces as SVG paths on top of the world map. This reuses the existing GeoJSON files.

**Option B — Combined regions TopoJSON:**
Pre-build a single world-regions TopoJSON combining all province data. One network request, but large file and build step required.

Recommend **Option A** for v1.1 — simpler, progressive loading, reuses existing files.

### GeoJSON availability

Province GeoJSON files exist in `client/public/geo/` for all current 30 T1+T2 countries (confirmed — they're used on country detail pages). New T2 countries from Stage 1A will need GeoJSON files added alongside their province seed data.

### City markers in regions mode

When regions mode is active, city dots are shown for countries where the user has visited cities. Dots use the city lat/lng added in Stage 1D. Visited cities show in a highlight colour; unvisited cities in the user's visited countries are greyed out.

## Implementation Order

1. **Stage 1B first (data foundation):** Add city data for T3 countries + add lat/lng to cities table. This unblocks 1C and 1D.
2. **Stage 1A:** Add province GeoJSON + seed data for new T2 countries, then reclassify them in `points.js`.
3. **Stage 1C/1D together:** City checking for T2 + cities on province maps (share the lat/lng migration).
4. **Stage 1E:** Quick-add from world map (self-contained UI change).
5. **Stage 2:** Visited regions map mode (depends on 1A and 1D for full effect).

## Open Questions

1. **Canada/Australia/Argentina/Kazakhstan/Algeria anomaly:** These are T3 but rank 2, 6, 8, 9, 10 globally by area — larger than most T1 countries. Should they be added to the Tier 2 promotion list even though they fall outside ranks 11–40?

2. **Libya province complexity:** Libya's administrative divisions are politically contested. Use 3 broad historical regions (Tripolitania, Cyrenaica, Fezzan) or skip provinces and give it cities only?

3. **T2 city + province cap:** Should combined province + city explorer points be hard-capped at the explorer ceiling for T2 countries?

4. **The 0.4 multiplier origin:** Confirm this is the intended city contribution weight for T2 (cities = 40% on top of provinces). Where was this calculated?

5. **Regions mode performance:** Fetching per-country GeoJSON for 30 T1/T2 countries on the global map could mean 30 network requests. Should we lazy-load only visited countries, or pre-bundle the most common ones?
